# Reproducibility Plan

Target reproducibility tier at bootstrap: `gate+expected`.

This project aims to make the following outputs reproducible from versioned inputs and declared configuration:

- surfaceome universe
- expression tables
- normal/selectivity/risk scores
- HPA protein evidence table
- topology and isoform table
- Balanced ranking
- sensitivity rankings
- tier assignments
- main figures
- supplementary tables

## Non-Negotiable Rules

1. Raw data in `data/raw/` is immutable after download.
2. Every downloaded file must be registered in `config/datasets.yaml`.
3. Every raw file must have a checksum recorded in `data/checksums/`.
4. Main results must be generated by `workflow/` or `src/`, not manual notebooks.
5. Changes that alter the top 20 ranking must be logged in `docs/design_decisions.md` and `docs/analytical_decisions_registry.md`.

## Current Bootstrap Scope

No raw data has been downloaded yet. No ranking has been generated. Controls and scoring scenarios are preregistered before data execution.

## Required Audit Before Submission

- clean clone or clean directory run
- environment recreated from lockfile
- `make smoke-test`
- full workflow run when implemented
- output comparison with declared tolerances
- release checklist in `release/release_checklist.md`
